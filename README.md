# Thailand Human Development Report 2009 — AI Extraction Pipeline

Local dual-LLM pipeline that extracts structured indicators, thematic distribution,
and narrative insights from the Thailand HDR 2009 PDF, with independent quality
evaluation and an interactive dashboard.

**Model A (extraction/summarization):** Llama 3.1
**Model B (evaluation):** Mistral — a genuinely different model, used to avoid
self-evaluation bias.

## Setup

```
pip install ollama pandas matplotlib streamlit plotly pymupdf --break-system-packages
ollama pull llama3.1
ollama pull mistral
ollama serve   # run in a separate terminal, leave it open
```

Place the source PDF at `data/nhdr2009thailand.pdf` before running anything.

## Run order

Run these in sequence from the project root. Each step reads the previous
step's output from `outputs/`, so order matters.

| # | Script | Reads | Writes | What it does |
|---|--------|-------|--------|---------------|
| 1 | `src/chunk_chapters.py` | `data/nhdr2009thailand.pdf` | `outputs/chapters.json` | Extracts PDF text, splits into 8 chapters using verified page-offset mapping from the Table of Contents |
| 2 | `src/summarize_chapters.py` | `outputs/chapters.json` | `outputs/chapter_summaries.json` | Summarizes each chapter (<100 words) with Llama 3.1; map-reduce for oversized chapters |
| 3 | `src/key_results.py` | `outputs/chapter_summaries.json` | `outputs/key_results.json` | Generates 5–8 key result bullets for the full report |
| 4 | `src/extract_indicators.py` | `outputs/chapters.json` | `outputs/indicators.json` | Extracts HDI rank, life expectancy, GNI per capita, population, etc. Requires verbatim source quotes and rejects hallucinated/ungrounded values via 4 verification checks |
| 5 | `src/thematic_extraction.py` | `outputs/chapters.json` | `outputs/theme_counts.json`, `outputs/strengths_challenges.json` | Counts passages per theme (education, health, inequality, economy, gender, climate, employment); extracts strengths/challenges |
| 6 | `src/evaluate_outputs.py` | `outputs/chapters.json`, `outputs/chapter_summaries.json`, `outputs/indicators.json` | `outputs/summary_evaluations.json`, `outputs/indicator_evaluation.txt` | Mistral evaluates Llama's chapter summaries (consistency/completeness/factual alignment) and reviews the indicator extraction reliability |
| 7 | `src/evaluate_task2.py` | `outputs/chapters.json`, `outputs/theme_counts.json`, `outputs/strengths_challenges.json` | `outputs/task2_evaluation.json` | Mistral evaluates theme count plausibility and strengths/challenges support |

After step 7, all required outputs exist in `outputs/`.

## Dashboards (run anytime after step 7)

**Static HTML** (no server, fully offline, Chart.js embedded inline):
Just open `dashboard.html` in any browser. Data is hardcoded from the
verified outputs above — will not auto-update if you re-run the pipeline
with different data (regenerate manually if needed).

**Interactive Streamlit version** (reads live from `outputs/` at runtime):
```
pip install streamlit plotly pandas
streamlit run dashboard.py
```

## Known limitations (documented, not bugs)

- `hdi_value`, `expected_years_of_schooling`, `mean_years_of_schooling` are
  correctly `null` — not present as extractable text in this report (HDI is
  shown only as a chart figure).
- `gni_per_capita` is sourced from a GDP-per-capita (PPP) statement in the
  text — related but not an identical measure to GNI.
- `life_expectancy_years` (70.6) is men-specific; the source separately
  states 77.5 years for women.
- Theme counts are approximate (chapter sub-chunking can occasionally split
  a passage across a chunk boundary).
- Task 2 evaluation (step 7) samples ~700 characters per chapter for the
  evaluator prompt, due to Ollama's context window limits — findings should
  be read as "no issues found in the sampled excerpt," not an exhaustive
  validation of the full extracted output.
- The population/poverty time-series shown in both dashboards is not yet
  generated by a script — it was manually extracted from Table 2.1 in the
  source PDF and hardcoded into both dashboard files.
